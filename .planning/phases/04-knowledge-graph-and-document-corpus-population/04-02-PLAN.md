  <action>
Create `scripts/seed-neo4j-analogues.py` that:

**Step 1: Load regime data and build feature vectors**
- Read `neo4j/seed/regime_data.json` (created in Plan 01)
- Extract FRED dimension vectors: [gdp_avg, cpi_avg, unrate_avg, fedfunds_avg]
- Handle null values: if any FRED dimension is null for a regime, exclude that regime from similarity computation with a warning log. Do NOT use 0 as a substitute for null — it would create false similarity.
- Build a numpy array of feature vectors for regimes with complete FRED data

**Step 2: Compute normalized pairwise cosine similarity**
- Use `sklearn.preprocessing.MinMaxScaler` to normalize each dimension to [0,1] range (per RESEARCH.md anti-pattern: FRED series have wildly different scales)
- Use `scipy.spatial.distance.cdist` with metric='cosine' to compute pairwise distance matrix
- Convert to similarity: `sim_matrix = 1 - distance_matrix`
- Log the similarity matrix shape and value range

**Step 3: Select top analogues per regime (threshold-filtered)**
- For each regime, rank all other regimes by similarity score (descending)
- Keep top 5 analogues that exceed the similarity threshold
- Start with threshold = 0.75 (Claude's discretion per CONTEXT.md — conservative to ensure enough analogues)
- If a regime has fewer than 2 analogues above threshold, log a warning but do NOT lower the threshold for just that regime — sparse connectivity is acceptable
- Build a list of analogue pairs: {from_id, to_id, similarity_score, dimensions_matched, period_start, period_end}
- `dimensions_matched` is always ["gdp", "cpi", "unrate", "fedfunds"] (the 4 FRED dimensions used)
- `period_start` and `period_end` come from the SOURCE regime (the one being compared FROM)

**Step 4: Generate static narratives via Gemini API**
- For each analogue pair, call Gemini (google-generativeai) to generate a 2-3 sentence narrative explaining WHY these two periods are similar
- Prompt template:
  ```
  You are a macroeconomic analyst. Compare these two economic periods:

  Period A: {name_a} ({period_start_a} to {period_end_a})
  - GDP growth: {gdp_a}%, CPI: {cpi_a}%, Unemployment: {unrate_a}%, Fed Funds: {fedfunds_a}%
  - Context: {notes_a}

  Period B: {name_b} ({period_start_b} to {period_end_b})
  - GDP growth: {gdp_b}%, CPI: {cpi_b}%, Unemployment: {unrate_b}%, Fed Funds: {fedfunds_b}%
  - Context: {notes_b}

  Cosine similarity: {similarity_score:.3f}

  In 2-3 sentences, explain the key macroeconomic parallels between these periods that make them useful historical analogues for investment analysis. Focus on policy environment, growth trajectory, and inflation dynamics.
  ```
- Use `gemini-2.0-flash` model (fast, cheap for batch generation)
- Add `time.sleep(1)` between calls to avoid rate limits (per RESEARCH.md Pitfall 6)
- Cache narratives to `neo4j/seed/analogue_narratives.json` — if this cache file exists on re-run, skip Gemini calls for pairs already cached. This prevents redundant API spend.
- On Gemini API error (429, 500, etc.): retry with exponential backoff (2s, 4s, 8s), max 3 retries. If all retries fail, set narrative to "Narrative generation failed — manual review needed" and continue.

**Step 5: MERGE HAS_ANALOGUE relationships into Neo4j**
- Use UNWIND+MERGE pattern (from RESEARCH.md Pattern 2):
  ```cypher
  UNWIND $batch AS row
  MATCH (src:Regime {id: row.from_id})
  MATCH (tgt:Regime {id: row.to_id})
  MERGE (src)-[rel:HAS_ANALOGUE]->(tgt)
  SET rel.similarity_score   = row.similarity_score,
      rel.dimensions_matched = row.dimensions_matched,
      rel.period_start       = row.period_start,
      rel.period_end         = row.period_end,
      rel.narrative          = row.narrative
  ```
- Use `SET rel.X = row.X` (not `SET rel += row`) to explicitly set each property — clearer than map merge for relationship properties
- Note: HAS_ANALOGUE is directional but we create relationships in BOTH directions (if A is analogue of B, B is analogue of A) — the Cypher query in Phase 5 uses directional traversal

**Step 6: Validate after write**
- Run validation query: `MATCH ()-[r:HAS_ANALOGUE]->() WHERE r.similarity_score IS NULL OR r.dimensions_matched IS NULL RETURN count(r)` — must return 0
- Run count query: `MATCH ()-[r:HAS_ANALOGUE]->() RETURN count(r)` — log total relationship count
- Print summary: "Created {N} HAS_ANALOGUE relationships across {M} regime pairs"
