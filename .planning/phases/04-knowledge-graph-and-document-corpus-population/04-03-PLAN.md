**Step 1: Create `data/fomc/manifest.json`**

Define ~10-15 FOMC minutes at key monetary policy turning points (per CONTEXT.md):
```json
[
  {
    "doc_id": "fomc_20081029",
    "url": "https://www.federalreserve.gov/monetarypolicy/files/fomcminutes20081029.pdf",
    "date": "2008-10-29",
    "title": "FOMC Minutes — October 2008 (Emergency Rate Cut to 1%)",
    "doc_type": "fomc_minutes",
    "regime_context": "gfc_credit_crisis"
  }
]
```

Include FOMC minutes from these key turning points:
- 2008 GFC emergency cuts (Oct 2008, Dec 2008)
- 2009 QE1 announcement context (Mar 2009)
- 2010 QE2 announcement (Nov 2010)
- 2013 taper tantrum (Jun 2013)
- 2015 first rate hike (Dec 2015)
- 2019 rate cut pivot (Jul 2019)
- 2020 COVID emergency (Mar 2020)
- 2021 inflation signals (Nov 2021)
- 2022 aggressive hikes (Jun 2022, Sep 2022)
- 2023 pause/hold (Jun 2023, Nov 2023)
- 2024 rate cut start (Sep 2024)

Note: Some URLs may use different patterns for older minutes. The script should handle HTTP 404 gracefully and log which documents could not be downloaded.

**Step 2: Create `data/sbv/manifest.json`**

Define ~20-30 SBV monetary policy documents (per CONTEXT.md). Since SBV documents are harder to download programmatically, this manifest serves as a registry. The script processes PDF files placed in `data/sbv/` directory matching manifest entries.

```json
[
  {
    "doc_id": "sbv_rate_2008q4",
    "filename": null,
    "date": "2008-12-01",
    "title": "SBV refinancing rate cut Q4 2008",
    "doc_type": "sbv_rate_decision",
    "lang": "en",
    "notes": "Place English PDF in data/sbv/ and update filename field"
  }
]
```

For SBV entries: set `filename` to null initially. The user will manually download documents from sbv.gov.vn/en and place them in `data/sbv/`, then update the manifest with filenames. The script skips entries with null filenames.

**Step 3: Create `scripts/seed-qdrant-macro-docs.py`**

Python script that:

1. **Downloads FOMC PDFs** from URLs in `data/fomc/manifest.json`
   - Save to `data/fomc/{doc_id}.pdf`
   - Skip if file already exists (cached download)
   - Handle HTTP errors gracefully: log warning and skip document
   - Use `httpx` for downloads (already in sidecar requirements)

2. **Reads SBV PDFs** from `data/sbv/` directory
   - Only process entries in manifest that have non-null filenames
   - Log count of skipped entries (null filename)

3. **Extracts text from PDFs** using `pdfplumber`
   - Handle extraction failures gracefully (corrupted PDFs, scanned images)
   - Log page count and character count per document

4. **Chunks text** using `langchain-text-splitters.RecursiveCharacterTextSplitter`
   - chunk_size=2048 characters (~512 tokens for bge-small)
   - chunk_overlap=256 characters (~12% overlap, per RESEARCH.md Pattern 4)
   - separators=["\n\n", "\n", ". ", " ", ""]

5. **Embeds chunks** with FastEmbed
   - `TextEmbedding(model_name="BAAI/bge-small-en-v1.5")`
   - batch_size=64
   - Log embedding generation time

6. **Upserts to Qdrant** macro_docs_v1 collection
   - Use deterministic UUIDs based on doc_id + chunk_index (uuid5 with namespace) — this ensures idempotent re-runs overwrite the same points
   - Payload per point:
     ```json
     {
       "text": "chunk text content",
       "source": "fomc|sbv",
       "doc_id": "fomc_20081029",
       "doc_type": "fomc_minutes|sbv_rate_decision|sbv_policy_report",
       "document_date": "2008-10-29",
       "title": "FOMC Minutes — October 2008",
       "chunk_index": 0,
       "total_chunks": 15,
       "regime_context": "gfc_credit_crisis"
     }
     ```
   - Use `client.upload_points()` with batch_size=64, wait=True

7. **Validates** after upsert
   - Query collection info: total point count
   - Run a test similarity search: "Federal Reserve rate decision" — log top-3 results with scores
   - Print summary: "Uploaded {N} chunks from {M} documents to macro_docs"
