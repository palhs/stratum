**Step 2: Create `scripts/seed-qdrant-earnings-docs.py`**

Python script that:

1. **Reads manifest** from `data/earnings/manifest.json`
   - Filter to entries with non-null filenames (skip undownloaded)
   - Group by ticker for progress reporting
   - Log: "Processing {N} documents for {M} tickers ({S} skipped — no file)"

2. **Reads document files** from `data/earnings/{ticker}/` directory
   - Support PDF files (via pdfplumber)
   - Support plain text files (.txt) — for pre-extracted or machine-translated content
   - Handle extraction failures gracefully (log warning, skip document)
   - For Vietnamese-only documents (lang="vi"): embed as-is but add payload flag `"lang": "vi"` and log warning about degraded embedding quality with bge-small-en-v1.5

3. **Chunks text** using `langchain-text-splitters.RecursiveCharacterTextSplitter`
   - chunk_size=2048 characters (~512 tokens)
   - chunk_overlap=256 characters (~12% overlap)
   - separators=["\n\n", "\n", ". ", " ", ""]
   - Same settings as seed-qdrant-macro-docs.py for consistency

4. **Embeds chunks** with FastEmbed
   - `TextEmbedding(model_name="BAAI/bge-small-en-v1.5")`
   - batch_size=64

5. **Upserts to Qdrant** earnings_docs_v1 collection
   - Deterministic UUIDs: uuid5 based on doc_id + chunk_index
   - Payload per point:
     ```json
     {
       "text": "chunk text content",
       "ticker": "VCB",
       "company_name": "Vietcombank",
       "fiscal_period": "2024-Q4",
       "fiscal_year": 2024,
       "doc_type": "quarterly_financial_statement",
       "doc_id": "vcb_2024q4",
       "lang": "en",
       "chunk_index": 0,
       "total_chunks": 8
     }
     ```
   - Use `client.upload_points()` with batch_size=64, wait=True

6. **Validates** after upsert
   - Query collection info: total point count
   - For each ticker that was processed: run a similarity search for the ticker symbol, log top result with score
   - Print summary: "Uploaded {N} chunks from {M} documents ({T} tickers) to earnings_docs"
