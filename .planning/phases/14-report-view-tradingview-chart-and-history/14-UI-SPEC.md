## Spacing Scale

Declared values (multiples of 4 only). Consistent with Phase 12/13 established usage.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (gap-1) |
| sm | 8px | Compact element spacing, badge internal padding (gap-2, p-2) |
| md | 16px | Default element spacing, card internal padding (p-4, gap-4) |
| lg | 24px | Section padding, vertical rhythm between report sections (py-6, gap-6) |
| xl | 32px | Layout gaps between major sections (gap-8) |
| 2xl | 48px | Major section breaks, chart top/bottom margin (py-12) |
| 3xl | 64px | Page-level vertical padding |

Exceptions:
- Floating bilingual toggle: fixed top-4 right-4 (16px from viewport edges) — uses md token
- Touch targets for timeline row click areas: min-h-[44px] — matches EmptyState button pattern from Phase 12
- Chart height (desktop): 400px fixed height — not a spacing token, declared separately in Component Inventory

Source: CONTEXT.md (layout decisions), existing code patterns in TickerCard.tsx, EmptyState.tsx

## Typography

Four roles, two weights. Consistent with existing dashboard component usage.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 24px (text-2xl) | Bold (700) | 1.2 | Hero tier badge text inside summary card |
| Heading | 20px (text-xl) | Bold (700) | 1.2 | Symbol heading ("Back to Dashboard" page title), section labels |
| Body | 16px (text-base) | Regular (400) | 1.5 | Full report markdown body text, verdict text |
| Label | 14px (text-sm) | Regular (400) | 1.4 | Sub-assessment labels (Macro/Valuation/Structure), history timeline metadata, dates, time elapsed, muted-foreground supplementary text |

Weights used: Regular (400) and Bold (700) only. Semibold (600) is NOT introduced — existing bold/regular pattern from Phase 12 is preserved.

All secondary metadata (time elapsed, supplementary context text) uses Label (14px) at text-muted-foreground to communicate hierarchy through color contrast rather than a separate type size.

Source: Existing TierBadge.tsx (text-2xl font-bold), TickerCard.tsx (text-xl font-bold, text-sm), CONTEXT.md

## Color

Semantic tokens from globals.css CSS variables. Tier colors from TierBadge.tsx TIER_STYLES — unchanged.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(var(--background)) — zinc-50 light / zinc-950 dark | Page background, report content area |
| Secondary (30%) | hsl(var(--card)) — white light / zinc-950 dark | Summary card surface, history timeline card rows |
| Accent (10%) | hsl(var(--primary)) — zinc-900 light / zinc-50 dark | Active/selected history row highlight border, expand button focus ring |
| Destructive | hsl(var(--destructive)) — red-500 | Downgrade arrow in history timeline (tier regression indicator) |

Accent reserved for:
1. Active/selected history timeline row — left border accent to indicate which historical report is displayed
2. Expand/collapse button hover state (button ring)
3. Focus-visible ring on interactive timeline rows (accessibility)

Tier-semantic colors (NOT general UI accent — reserved exclusively for tier indication):
- Favorable: bg-teal-100 text-teal-800 (light) / bg-teal-900/30 text-teal-300 (dark)
- Neutral: bg-slate-100 text-slate-700 (light) / bg-slate-800 text-slate-300 (dark)
- Cautious: bg-amber-100 text-amber-800 (light) / bg-amber-900/30 text-amber-300 (dark)
- Avoid: bg-rose-100 text-rose-800 (light) / bg-rose-900/30 text-rose-300 (dark)

Assessment change arrow colors:
- Upgrade arrow (tier improvement): text-teal-600 dark:text-teal-400 — matches STEP_LABELS completed check color from StepList.tsx
- Downgrade arrow (tier regression): text-rose-600 dark:text-rose-400 — consistent with Avoid tier rose palette
- No change: no arrow rendered (do not use a neutral dash icon; absence communicates stability)

Source: globals.css, TierBadge.tsx, StepList.tsx, CONTEXT.md

## Component Inventory

New components required for this phase. All existing components (TierBadge, Badge, Card, Button, Skeleton) reused without modification.

### ReportSummaryCard
- Container: Card component (shadcn), full-width, p-4 spacing
- Hero element: TierBadge at display size (text-2xl font-bold) — reuse existing component directly
- Sub-assessments: Three inline labels (Macro / Valuation / Structure) each showing sub-tier value — text-sm, muted-foreground for label, foreground for value
- Verdict: One-line text at body size (text-base), foreground color, line-height 1.5
- Expand/collapse trigger: Button variant="ghost" at bottom of card, full-width, label "Read full report" / "Collapse report" — chevron icon (lucide ChevronDown/ChevronUp)
- Collapsed height: natural card height with summary content (no fixed height clipping)
- Expanded: full report section appears below chart in page flow — card does not grow

### BilingualToggle
- Floating position: fixed top-4 right-4 (z-index above page content)
- Implementation: Two-segment button group — "VI" | "EN" — using shadcn Button variant="outline" with active segment showing variant="default"
- Size: text-sm, px-3 py-1.5 (compact, not full touch target — this is a secondary persistent control)
- State persisted: localStorage key "stratum-report-lang", values "vi" | "en"
- Default: "vi" (Vietnamese primary per PROJECT.md)
- Transition: instant switch — no loading state (both content versions fetched upfront)

### TradingViewChart
- Loaded via: dynamic({ ssr: false }) — established decision from Phase 12, STATE.md
- Library: lightweight-charts (TradingView Lightweight Charts — already in use per CONTEXT.md getOhlcv pattern)
- Chart type: Candlestick series (weekly OHLCV)
- Overlays: MA50 (line series, blue) and MA200 (line series, orange) — visible by default
- Volume: Histogram below candlestick area — muted color (volume bars use rgba not solid colors to avoid visual competition)
- Default range: 52 weeks (1 year) — matches OHLCVPoint data already fetched by getOhlcv()
- Interaction: zoomable and pannable (Lightweight Charts default behavior — no additional config needed)
- Height: 400px on desktop (>=768px), 260px on mobile (<768px) — responsive via CSS
- Width: 100% of container (no fixed width)
- Container class: w-full rounded-lg overflow-hidden border border-border
- SSR safety: wrapped in dynamic import with loading fallback (Skeleton at same height)

### ReportMarkdown
- Renderer: react-markdown (to be installed) with remark-gfm plugin
- Container: prose class via @tailwindcss/typography plugin (to be installed) — scoped to max-w-none to fill column
- Language switching: conditional render of vi vs en markdown content — no animation, instant swap
- Code blocks: inherit prose styling — no syntax highlighting required (reports do not contain code)
- Dark mode: prose-invert class applied when dark mode active

### HistoryTimeline
- Container: vertical list (ul), no card wrapper — full-width below chart+report sections
- Section heading: "Report History" at heading size (text-xl font-bold), mb-4
- Each history row:
  - Layout: horizontal flex, items-center, gap-3, py-3, border-b border-border
  - Min height: 44px (touch target compliance)
  - Date: text-sm text-muted-foreground, fixed width ~80px
  - TierBadge: existing component, smaller variant — text-sm font-semibold (override display size for timeline density)
  - Verdict snippet: text-sm text-foreground, truncated with line-clamp-1, flex-1
  - Change arrow: lucide ArrowUp (teal-600) / ArrowDown (rose-600) / no element — rightmost, w-4 h-4
  - Active row: left border 2px solid hsl(var(--primary)), bg-muted/50
  - Hover: bg-muted/30 cursor-pointer
  - Focus-visible: ring-2 ring-ring (accessibility)
- "Load more" button: Button variant="outline", full-width, mt-4, shown only when total > loaded count
- Loading skeleton: 3 rows of Skeleton at 44px height with gap-2

### ReportPageSkeleton
- Mirrors page layout in skeleton form
- Summary card skeleton: Skeleton w-full h-32 rounded-lg
- Chart skeleton: Skeleton w-full h-[400px] md:h-[400px] h-[260px] rounded-lg
- Timeline skeleton: 3x Skeleton h-11 w-full rounded

## Page Layout Contract

### /reports/[symbol] — Single-column vertical stack

```
[Back to Dashboard]           ← Button variant="ghost", text-sm, mb-4
[Symbol heading]              ← text-xl font-bold
─────────────────────────────
[ReportSummaryCard]           ← tier hero + sub-assessments + verdict + expand button
─────────────────────────────
[TradingViewChart]            ← 400px desktop / 260px mobile, mt-6
─────────────────────────────
[ReportMarkdown]              ← rendered only when expanded, mt-6, animate-in fade-in duration-200
─────────────────────────────
[HistoryTimeline]             ← mt-8, "Report History" heading + rows
─────────────────────────────
                              ← [BilingualToggle] FIXED top-4 right-4
```

Max content width: max-w-4xl mx-auto (narrower than dashboard max-w-7xl — report is text-primary content).
Page padding: px-4 py-6 (mobile), px-6 py-8 (desktop).

### Expand/Collapse behavior
- Default state: summary card visible, ReportMarkdown NOT rendered (DOM not mounted)
- On "Read full report": ReportMarkdown mounts below chart — fade-in transition (animate-in fade-in duration-200 from tailwindcss-animate)
- Summary card + chart remain visible above when expanded — user has price context while reading
- Expand/collapse state: local useState — not persisted (intentional, matches success criterion reading flow)

### History row click behavior
- Clicking a history row: replaces summary card content + report markdown in-place
- URL remains /reports/{symbol} (no navigation — state-managed in client component)
- Selected row gains active styling immediately on click — no loading delay visible at row level
- Report content area shows Skeleton while new report content loads from API

## Accessibility Contract

- All interactive elements have focus-visible:ring-2 ring-ring — consistent with existing TickerCard pattern
- History timeline rows: role="button" or anchor elements with aria-label including tier and date
- Upgrade/downgrade arrows: aria-label with full tier transition description (see Copywriting Contract)
- TradingView chart container: aria-label="Price chart for {SYMBOL}" — chart itself is canvas, not accessible to screen readers; label describes the container
- BilingualToggle: aria-label="Report language" on group, aria-pressed on each button
- Expand/collapse button: aria-expanded={isExpanded} on trigger button
- ReportMarkdown: wrapped in article element with lang attribute matching selected language ("vi" or "en")
