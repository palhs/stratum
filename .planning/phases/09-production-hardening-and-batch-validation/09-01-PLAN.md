  <action>
    Create `scripts/batch-validate.py` — a standalone Python script that validates the 20-stock batch workload.

    **Script interface:**
    ```
    python scripts/batch-validate.py --base-url http://localhost:8001 [--timeout 600]
    ```

    **Dependencies:** `httpx` (already in project dependencies). Uses `subprocess` for `docker stats`/`docker inspect` (stdlib).

    **Implementation:**

    1. **Constants:**
       ```python
       VN30_TICKERS = [
           "VHM", "VNM", "VCB", "BID", "VPB",
           "TCB", "MBB", "CTG", "HPG", "MSN",
           "VIC", "GAS", "SAB", "FPT", "MWG",
           "REE", "VJC", "HDB", "STB", "ACB",
       ]
       SERVICES = [
           "stratum-postgres-1",
           "stratum-neo4j-1",
           "stratum-qdrant-1",
           "stratum-reasoning-engine-1",
       ]
       ```

    2. **`wait_for_health(base_url, timeout=60)`** — Poll GET /health until 200 or timeout. Print "Waiting for reasoning-engine health..." on retry. Raise SystemExit on timeout.

    3. **`submit_and_poll(client, base_url, ticker, timeout)`** — POST /reports/generate with `{"ticker": ticker, "asset_type": "equity"}`. Assert 202 response. Extract job_id. Poll GET /reports/{job_id} every 5 seconds until status is "completed" or "failed". Return `(ticker, job_id, status, elapsed_seconds)` tuple. Raise TimeoutError if job does not settle within timeout.

    4. **`capture_docker_stats()`** — Run `docker stats --no-stream --format "table {{.Name}}\t{{.MemUsage}}\t{{.MemPerc}}"` via subprocess.run(capture_output=True). Return stdout as string. Print the table to console.

    5. **`check_oom_status(services)`** — For each service name, run `docker inspect <name> --format "{{.State.OOMKilled}}"` via subprocess. Parse the boolean result. Return dict `{service_name: bool}`. Print per-service OOM status.

    6. **`run_batch(base_url, timeout_per_job)`** — Main orchestrator:
       - Call `wait_for_health(base_url)`
       - Print "Starting batch validation: 20 VN30 tickers (sequential)"
       - Capture initial docker stats snapshot
       - For each ticker in VN30_TICKERS:
         - Call `submit_and_poll()` — print ticker, job_id, status, elapsed time per job
         - Capture intermediate docker stats every 5 tickers
       - Capture final docker stats snapshot
       - Call `check_oom_status()` for all SERVICES
       - Print summary table: ticker | job_id | status | elapsed_seconds
       - Count completed, failed, oom_killed
       - Print PASS if oom_killed == 0, FAIL if any OOM detected
       - Return exit code 0 on PASS, 1 on FAIL

    7. **`main()`** with argparse:
       - `--base-url` (default: `http://localhost:8001`)
       - `--timeout` per-job timeout in seconds (default: 600)
       - Call run_batch(), sys.exit with return code

    **Error handling:**
    - Connection refused on health check: print clear message "Cannot connect to reasoning-engine at {base_url}. Is the Docker stack running?"
    - 409 on POST: log and skip (should not happen with sequential processing of distinct tickers, but handle gracefully)
    - Job timeout: log as "TIMEOUT" status in summary table, continue to next ticker

    **Output format:**
    ```
    ============================================================
    Batch Validation Summary
    ============================================================
    Ticker  | Job ID | Status    | Elapsed (s)
    --------|--------|-----------|------------
    VHM     | 1      | completed | 45
    VNM     | 2      | completed | 38
    ...
    ============================================================
    Completed: 18  Failed: 2  OOM Kills: 0
    Result: PASS (no OOM kills detected)
    ============================================================
    ```
