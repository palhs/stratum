## Color

All values expressed as shadcn CSS custom property references. Hex equivalents provided for light mode.

| Role | CSS Variable | Light Mode Value | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` — white | Page background, card surfaces |
| Secondary (30%) | `--card` / `--muted` | `hsl(0 0% 100%)` / `hsl(240 4.8% 95.9%)` | Cards, muted text areas |
| Accent (10%) | `--primary` | `hsl(240 5.9% 10%)` — near-black | GenerateButton (secondary variant border + text only) |
| Destructive | `--destructive` | `hsl(0 72.2% 50.6%)` — red | Failed step icon (XCircle), error toast |

Accent reserved for: GenerateButton border/text (secondary variant), focus ring on card link, active step pulsing icon color.

**Semantic step status colors (not shadcn tokens — explicit Tailwind classes):**

| Step Status | Icon | Color Class | Purpose |
|-------------|------|-------------|---------|
| completed | Check | `text-teal-600 dark:text-teal-400` | Completed pipeline node — matches Favorable tier green |
| in_progress | Loader2 (animated spin) | `text-foreground` | Currently executing node |
| pending | Circle (outline) | `text-muted-foreground` | Not yet reached node |
| failed | XCircle | `text-destructive` | Node that errored |

Source: Tier badge colors established in Phase 12 (`TierBadge.tsx` — teal for Favorable). Completed step uses teal to signal "this step went well" without introducing a new color token. Destructive red already defined in globals.css.

## Interaction States

### GenerateButton

| State | Visual |
|-------|--------|
| Default | Secondary variant button, full card width, 44px height, label "Generate Report" |
| Hover | shadcn secondary hover (muted background darkens slightly) |
| Disabled (generating) | `disabled` attribute + `opacity-50 cursor-not-allowed` — cannot be clicked |
| Hidden | Not rendered — replaced by StepList when `isGenerating === true` |

### TickerCard during generation

| Element | Behavior |
|---------|----------|
| Tier badge | Remains visible at top of card — no change |
| Ticker symbol + name | Remains visible |
| Sparkline | Hidden (`hidden` class) — replaced by StepList |
| Last report date line | Hidden (`hidden` class) — replaced by StepList |
| GenerateButton | Hidden — replaced by StepList |
| StepList | Shown — card height expands in place via Tailwind transition on height (`transition-all duration-300`) |
| Card link (navigate to /reports/{symbol}) | Remains functional — clicking the card header area navigates |

### Card expand/collapse animation

- Transition: `transition-all duration-300 ease-in-out` on the collapsible section wrapper
- Expand: StepList section fades in + height grows from 0 to auto (use `grid grid-rows-[0fr]` / `grid-rows-[1fr]` pattern for smooth height animation — avoids JS-measured height)
- Collapse: Reversed on success or 3-second error delay

### Step list items

| Transition | Spec |
|------------|------|
| Step status change (pending -> in_progress) | No transition — immediate icon swap |
| Step status change (in_progress -> completed) | Loader2 replaced by Check — no transition |
| Step status change (in_progress -> failed) | Loader2 replaced by XCircle — no transition |

Note: Instant icon swap (no crossfade) — reduces complexity; the Loader2 spin animation itself provides sufficient motion feedback for the active step.

### Loader2 spin

- Icon: `lucide-react` Loader2
- Animation: `animate-spin` (Tailwind built-in, 1s linear infinite)
- Color: `text-foreground`

## SSE Event-to-UI Mapping

This section documents the visual contract for each SSE event type so the executor can implement without ambiguity.

| SSE Event | `event_type` | UI Action |
|-----------|-------------|-----------|
| node_start | `"node_start"` | Set `steps[node] = 'in_progress'` — show Loader2 spin |
| node_complete (no error) | `"node_complete"`, error: null | Set `steps[node] = 'completed'` — show Check in teal |
| node_complete (with error) | `"node_complete"`, error: string | Set `steps[node] = 'failed'` — show XCircle in destructive red |
| complete (SSE terminal event) | n/a (EventSource named event: `complete`) | Close EventSource, collapse card after 0ms delay, refresh ticker data, show success toast |
| SSE onerror | n/a (browser EventSource error callback) | Close EventSource, hold card expanded 3–4 seconds so user sees failed step, then collapse, show error toast |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| GenerateButton disabled state | `disabled` HTML attribute — screen readers announce "Generate Report, dimmed" |
| Step list | `<ul>` / `<li>` semantics — screen readers enumerate steps |
| Step icon aria-label | Each StepIcon includes `aria-label` matching step status: "Macro Analysis — completed", "Valuation — in progress", etc. |
| Loader2 animation | `aria-live="polite"` region wrapping StepList — screen readers announce step transitions without interrupting user |
| Card link during generation | Card `<Link>` retains `href` and is focusable — keyboard users can still navigate to report page |
| Focus ring | `focus-visible:ring-2 focus-visible:ring-ring` on card link (inherited from Phase 12) |
