    **CRITICAL BLOCKER RESOLUTION:** Existing Qdrant collections use unnamed single-vector config. LlamaIndex QdrantVectorStore with `enable_hybrid=True` requires named vectors `text-dense` (384-dim Cosine) and `text-sparse` (sparse, Dot with IDF modifier). Collections must be recreated with correct config.

    **scripts/init-qdrant.sh changes:**
    Update `create_collection_if_missing()` to create collections with named vector config for macro_docs_v1 and earnings_docs_v1. The three embedding collections (macro_embeddings_v1, valuation_embeddings_v1, structure_embeddings_v1) are NOT used by Phase 5 retrieval and should remain unchanged for now.

    For macro_docs_v1 and earnings_docs_v1, change the collection creation payload from:
    ```json
    {"vectors": {"size": 384, "distance": "Cosine"}}
    ```
    to:
    ```json
    {
      "vectors": {
        "text-dense": {"size": 384, "distance": "Cosine"}
      },
      "sparse_vectors": {
        "text-sparse": {"modifier": "idf"}
      }
    }
    ```

    Create a NEW helper function `create_hybrid_collection_if_missing()` for the two document corpus collections that uses the named-vector payload. Keep the original `create_collection_if_missing()` unchanged for the three embedding collections.

    **IMPORTANT:** Since init-qdrant.sh only creates if missing, the existing collections must be deleted first for recreation. Add a `recreate_hybrid_collection()` function that:
    1. Deletes the collection if it exists (curl -X DELETE)
    2. Creates with new named-vector config
    3. Creates alias
    Log clearly: "Recreating {collection} with named vector config for hybrid search"

    **scripts/seed-qdrant-macro-docs.py changes:**
    Update the upload_points call to use named vectors. Change from:
    ```python
    PointStruct(id=point_id, vector=dense_vector, payload=payload)
    ```
    to:
    ```python
    PointStruct(id=point_id, vector={"text-dense": dense_vector}, payload=payload)
    ```
    Do NOT add BM25 sparse vectors in the seed script. The LlamaIndex QdrantVectorStore handles sparse vector generation transparently at query time when `fastembed_sparse_model="Qdrant/bm25"` is configured. The sparse vectors will be generated and stored at query time by LlamaIndex.

    **WAIT — research correction:** Actually, for hybrid search to work, the sparse vectors must exist at index time, not query time. LlamaIndex only generates the sparse QUERY vector at query time. The index-time sparse vectors must be pre-computed.

    **Revised approach for seed scripts:**
    Add BM25 sparse vector generation using fastembed:
    ```python
    from fastembed import SparseTextEmbedding
    sparse_model = SparseTextEmbedding(model_name="Qdrant/bm25")
    ```
    For each chunk, compute both dense and sparse embeddings:
    ```python
    sparse_embeddings = list(sparse_model.embed([chunk_text]))
    sparse_vector = sparse_embeddings[0]
    # Convert to Qdrant SparseVector format
    from qdrant_client.models import SparseVector
    sparse_vec = SparseVector(indices=sparse_vector.indices.tolist(), values=sparse_vector.values.tolist())

    PointStruct(
        id=point_id,
        vector={"text-dense": dense_vector, "text-sparse": sparse_vec},
        payload=payload
    )
    ```

    **scripts/seed-qdrant-earnings-docs.py changes:**
    Same pattern as macro_docs — update to named vectors with both text-dense and text-sparse.

    Both seed scripts should initialize the sparse model once at the top alongside the dense model, and compute both embeddings per chunk. The sparse model download is lightweight (~2MB for BM25).

    After updating, the user will need to:
    1. Run `docker compose up qdrant-init` (recreates collections with hybrid config)
    2. Re-run seed scripts to populate with named vectors + BM25 sparse vectors
