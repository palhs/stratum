  <action>
    Create reasoning/app/retrieval/qdrant_retriever.py with two retrieval functions and collection-specific configuration.

    **Architecture decision (Claude's discretion — metadata filter pattern):**
    Create separate retriever instances per (collection, language) combination at function call time. Language filter is applied at the QdrantVectorStore level via `qdrant_filters` constructor parameter, NOT per-query (per Open Question #2 from research — constructor-level is the reliable pattern).

    **Connection setup:**
    - Read QDRANT_HOST, QDRANT_PORT, QDRANT_API_KEY from env vars (defaults: localhost, 6333, None)
    - Module-level QdrantClient creation (reusable across calls)
    - Use QdrantClient(host=..., port=..., api_key=..., https=False)

    **Collection-specific dense/sparse weights (locked decision: collection-specific tuning):**
    - macro_docs_v1: alpha=0.7 (favors dense semantic similarity — FOMC policy language is semantically rich)
    - earnings_docs_v1: alpha=0.5 (balanced — earnings docs have both keyword-heavy numbers and narrative)
    - Alpha controls dense vs sparse weight in Relative Score Fusion (1.0 = dense only, 0.0 = sparse only)

    **Function 1: search_macro_docs**
    ```python
    def search_macro_docs(
        query: str,
        lang: str = "en",
        top_k: int = 5,
        now_override: Optional[datetime] = None,
        client: Optional[QdrantClient] = None,
    ) -> list[DocumentChunk]:
    ```
    - Creates QdrantVectorStore with enable_hybrid=True, fastembed_sparse_model="Qdrant/bm25"
    - Dense vector name: "text-dense" (matching init-qdrant.sh)
    - Sparse vector name: "text-sparse" (matching init-qdrant.sh)
    - Language filter: FieldCondition(key="lang", match=MatchValue(value=lang))
    - Creates VectorStoreIndex.from_vector_store with FastEmbedEmbedding(model_name="BAAI/bge-small-en-v1.5")
    - Retriever with similarity_top_k=top_k, sparse_top_k=top_k*2, vector_store_query_mode="hybrid"
    - Converts LlamaIndex NodeWithScore to DocumentChunk
    - Applies freshness check on document_date from payload (threshold: 45 days)

    **Function 2: search_earnings_docs**
    ```python
    def search_earnings_docs(
        query: str,
        ticker: Optional[str] = None,
        lang: str = "en",
        top_k: int = 5,
        now_override: Optional[datetime] = None,
        client: Optional[QdrantClient] = None,
    ) -> list[DocumentChunk]:
    ```
    - Same hybrid setup as macro_docs but with alpha=0.5
    - Additional filter: if ticker provided, add FieldCondition(key="ticker", match=MatchValue(value=ticker))
    - Combines language and ticker filters with must=[...] compound filter
    - Freshness threshold: 120 days (quarterly)

    **LlamaIndex to DocumentChunk conversion:**
    ```python
    def _node_to_chunk(node_with_score, warnings) -> DocumentChunk:
        return DocumentChunk(
            id=node_with_score.node.id_,
            text=node_with_score.node.text,
            score=node_with_score.score,
            source=node_with_score.node.metadata.get("source", ""),
            lang=node_with_score.node.metadata.get("lang", ""),
            metadata=node_with_score.node.metadata,
            warnings=warnings,
        )
    ```

    **Error handling:**
    - If retriever returns 0 results, raise NoDataError with descriptive message including query and filters
    - Transient Qdrant failures (connection timeout): let exception propagate (Claude's discretion on retry — defer retry logic to Phase 6 LangGraph checkpoint/retry, which handles node-level retries natively)

    **Logging:** INFO level per call: function name, query (first 100 chars), collection, lang, result count, elapsed_ms.

    **Update reasoning/app/retrieval/__init__.py:**
    Add exports for search_macro_docs, search_earnings_docs.

    **IMPORTANT ANTI-PATTERNS:**
    - Do NOT attempt dense-only fallback if hybrid fails — the collection MUST have sparse vectors (Plan 01 ensures this)
    - Do NOT use QdrantVectorStore without named vectors — collection config from Plan 01 uses text-dense/text-sparse
    - Do NOT skip language filter — bge-small-en-v1.5 produces degraded Vietnamese embeddings (Phase 4 flagged this)
