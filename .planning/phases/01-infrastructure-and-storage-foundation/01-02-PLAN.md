   SELECT 'CREATE DATABASE n8n_meta'
   WHERE NOT EXISTS (SELECT FROM pg_database WHERE datname = 'n8n_meta')\gexec

Node uniqueness constraints (available in Community Edition):
```cypher
// Switch to user database for node constraints
:use neo4j;

CREATE CONSTRAINT regime_id_unique IF NOT EXISTS
FOR (r:Regime) REQUIRE r.id IS UNIQUE;

CREATE CONSTRAINT time_period_id_unique IF NOT EXISTS
FOR (t:TimePeriod) REQUIRE t.id IS UNIQUE;
```

**File: `neo4j/init/02_apoc_triggers.cypher`**

APOC trigger for RESEMBLES relationship property enforcement (Community Edition workaround — see RESEARCH.md Pitfall 1):

```cypher
// MUST run against system database (Neo4j 5.x requirement)
// Enforces that all RESEMBLES relationships carry:
//   - similarity_score (FLOAT)
//   - dimensions_matched (LIST<STRING>)
//   - period (STRING)
// This is required by MACRO-04 and Phase 1 Success Criterion #3.
// Neo4j Community Edition does NOT support relationship property existence constraints.
// APOC Core trigger is the verified workaround.

CALL apoc.trigger.install(
  'neo4j',
  'enforce_resembles_properties',
  'UNWIND [rel IN $createdRelationships WHERE type(rel) = "RESEMBLES"] AS r
   WITH r
   WHERE r.similarity_score IS NULL OR r.dimensions_matched IS NULL OR r.period IS NULL
   CALL apoc.util.validate(
     true,
     "RESEMBLES relationship missing required properties: similarity_score, dimensions_matched, period. Got: similarity_score=%s, dimensions_matched=%s, period=%s",
     [r.similarity_score, r.dimensions_matched, r.period]
   )
   RETURN null',
  {phase: 'before'}
);

CALL apoc.trigger.start('neo4j', 'enforce_resembles_properties');
```

**Critical notes from RESEARCH.md:**
- `apoc.trigger.install()` is the Neo4j 5.x API — do NOT use `apoc.trigger.add()` (4.x, removed)
- First argument to `apoc.trigger.install()` is the target database name ('neo4j')
- Must be called from system database context
- `apoc.trigger.start()` activates the trigger after installation
- `IF NOT EXISTS` is not available for triggers — the script should be idempotent by using `apoc.trigger.drop()` before install, or by accepting that re-running against an existing trigger updates it

**Make the trigger script idempotent** by adding a drop before install:
```cypher
CALL apoc.trigger.drop('neo4j', 'enforce_resembles_properties');
// ... then install
```
Wrap in a try-catch if available, or accept that drop fails silently on first run.

  <name>Task 3: Create Qdrant collection initialization script with alias versioning</name>
  <files>scripts/init-qdrant.sh</files>
  <action>
Create a shell script that initializes Qdrant collections using the alias versioning pattern from RESEARCH.md Pattern 7.

**File: `scripts/init-qdrant.sh`**

```bash
#!/bin/sh
set -e

QDRANT_URL="http://${QDRANT_HOST:-qdrant}:${QDRANT_PORT:-6333}"
AUTH_HEADER=""
if [ -n "$QDRANT_API_KEY" ]; then
  AUTH_HEADER="-H \"api-key: ${QDRANT_API_KEY}\""
fi

# Wait for Qdrant to be ready (belt-and-suspenders — health check should handle this)
echo "Waiting for Qdrant..."
until curl -sf "${QDRANT_URL}/healthz" > /dev/null 2>&1; do
  sleep 2
done
echo "Qdrant is ready."

# Create versioned collection: macro_embeddings_v1
# Vector size: 384 — matches FastEmbed default model (BAAI/bge-small-en-v1.5)
# This is Claude's discretion per CONTEXT.md. If embedding model changes in later phases,
# recreate the collection and update the alias — that's what the alias pattern is for.
# Distance: Cosine (standard for semantic similarity)

echo "Creating collection macro_embeddings_v1..."
curl -sf -X PUT "${QDRANT_URL}/collections/macro_embeddings_v1" \
  -H 'Content-Type: application/json' \
  ${AUTH_HEADER} \
  --data-raw '{
    "vectors": {
      "size": 384,
      "distance": "Cosine"
    }
  }' || echo "Collection may already exist, continuing..."

# Create stable alias pointing to versioned collection
echo "Creating alias macro_embeddings -> macro_embeddings_v1..."
curl -sf -X POST "${QDRANT_URL}/collections/aliases" \
  -H 'Content-Type: application/json' \
  ${AUTH_HEADER} \
  --data-raw '{
    "actions": [
      {"create_alias": {"collection_name": "macro_embeddings_v1", "alias_name": "macro_embeddings"}}
    ]
  }' || echo "Alias may already exist, continuing..."

echo "Qdrant initialization complete."
```

**Key decisions (Claude's discretion per CONTEXT.md):**
- Vector size 384 (FastEmbed's default BAAI/bge-small-en-v1.5) instead of 1536 (OpenAI) — RESEARCH.md suggested 1536 as placeholder, but FastEmbed is the project's chosen embedding approach (mentioned in ROADMAP Phase 3 success criteria). 384 is more memory-efficient for the 8GB VPS.
- Cosine distance metric (standard for semantic similarity)
- Script is idempotent — `|| echo "... may already exist"` handles re-runs
- Uses `sh` not `bash` since the curlimages/curl container uses alpine/sh

**Make the script executable:**
```bash
chmod +x scripts/init-qdrant.sh
```
