  <action>
Create `sidecar/app/services/markers_service.py` with the structure marker computation logic.

**Core function: `compute_and_upsert_markers(db_session, asset_types=None)`**

1. **Read source data from PostgreSQL:**
   - For stocks (asset_type='stock'): SELECT symbol, resolution, close, data_as_of FROM stock_ohlcv WHERE resolution='weekly' ORDER BY symbol, data_as_of
   - For gold spot (asset_type='gold_spot'): SELECT 'XAU' as symbol, 'weekly' as resolution, price_usd as close, data_as_of FROM gold_price ORDER BY data_as_of
   - For gold ETF (asset_type='gold_etf'): SELECT ticker as symbol, resolution, close, data_as_of FROM gold_etf_ohlcv WHERE resolution='weekly' ORDER BY ticker, data_as_of
   - Load into pandas DataFrames

2. **Compute markers per (symbol, resolution) group:**

   For each group sorted by data_as_of ASC:

   a. **Moving averages** (Claude's discretion on MA set — research recommends 10w, 20w, 50w):
      - ma_10w = close.rolling(10, min_periods=8).mean()  — 10 weeks ~ 2.5 months (short-term trend)
      - ma_20w = close.rolling(20, min_periods=16).mean() — 20 weeks ~ 5 months (medium trend)
      - ma_50w = close.rolling(50, min_periods=40).mean() — 50 weeks ~ 1 year (structural trend)

   b. **Full-history ATH drawdown:**
      - ath = close.expanding().max()
      - drawdown_from_ath = (close / ath) - 1.0

   c. **52-week high drawdown** (locked decision: compute BOTH):
      - high_52w = close.rolling(52, min_periods=26).max()
      - drawdown_from_52w_high = (close / high_52w) - 1.0

   d. **Valuation percentile (close price percentile rank):**
      - For stocks: window = 260 bars (5 years × 52 weeks), min_periods=52
      - For gold: window = 520 bars (10 years × 52 weeks), min_periods=104
      - close_pct_rank = close.rolling(window, min_periods=min_periods).apply(lambda x: pd.Series(x).rank(pct=True).iloc[-1])
      - Alternative (faster): close.rolling(window, min_periods).rank(pct=True) — use the simpler rolling rank

   e. **P/E percentile rank** (stocks only, gold = NULL):
      - If stock: join with stock_fundamentals on symbol to get latest pe_ratio, compute pe_pct_rank similarly
      - If gold: pe_pct_rank = NULL

3. **Assign asset_type column:** 'stock' for VN30, 'gold_spot' for XAU, 'gold_etf' for GLD

4. **Upsert to structure_markers:** INSERT ON CONFLICT (symbol, resolution, data_as_of) DO UPDATE all computed columns + ingested_at=NOW()

5. **Return total rows written**

**Recompute strategy (Claude's discretion):** Full recompute on each run. At VN30 scale (~30 symbols × 260 weeks = ~7,800 rows per resolution), this takes < 5 seconds. Incremental adds complexity with no meaningful performance gain.

**Edge case handling:**
- If stock_ohlcv has fewer rows than min_periods, MAs/drawdowns will be NULL — this is correct behavior, not an error
- Log NULL counts per column after computation as a health metric
- NEVER skip computation if data is insufficient — write rows with NULL markers, let downstream nodes handle
