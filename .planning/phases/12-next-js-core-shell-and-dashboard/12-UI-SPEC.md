# Phase 12 — UI Design Contract

> Visual and interaction contract for the Next.js Core Shell and Dashboard.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

<user_constraints>
## User Constraints (from CONTEXT.md)

### Locked Decisions

**Dashboard card layout**
- Responsive grid: 3 columns on desktop, 2 on tablet, 1 on mobile
- Cards reflow naturally — max 30 tickers in watchlist, all visible without pagination
- Cards are clickable — clicking navigates to `/reports/{symbol}` (placeholder page until Phase 14)

**Card visual hierarchy**
- Tier badge is the dominant/hero element — large, color-coded, centered on the card
- Visual order top-to-bottom: symbol + company name → tier badge (large) → sparkline → last report date
- Tier badge color scheme: muted/professional tones (teal for Favorable, slate for Neutral, warm amber for Cautious, muted rose for Avoid) — not traffic-light green/red

**Sparkline rendering**
- Simple inline SVG polyline from 52 weekly close prices — zero dependencies
- No axes, labels, tooltips, or interactivity — purely static visual indicator
- Color: green if price up year-over-year, red if down
- Full interactive chart is deferred to Phase 14 (TradingView Lightweight Charts)

**Empty state**
- "Your watchlist is empty" message with prompt to add tickers
- Show 3-5 suggested popular tickers as quick-add buttons (VNM, FPT, HPG, GLD, MWG)
- Quick-add calls PUT /watchlist to add the ticker

**Loading state**
- Skeleton cards with shimmer animation in the same responsive grid layout
- User immediately sees the dashboard structure before data loads

**Error state**
- Toast notification (non-blocking) for API errors
- First load failure: centered error message with Retry button
- Refresh failure: show stale cached data + toast "Refresh failed"

### Claude's Discretion
- CSS framework / component library choice (Tailwind, shadcn/ui, etc.)
- Next.js app router structure and route organization
- Auth middleware implementation (Next.js middleware vs route guards)
- Toast library choice
- Skeleton shimmer implementation approach
- Docker configuration details (multi-stage build, standalone output)

### Deferred Ideas (OUT OF SCOPE)
None — discussion stayed within phase scope
</user_constraints>

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal horizontal padding adjustment |
| sm | 8px | Compact element spacing, gap between symbol and company name text |
| md | 16px | Default card internal padding, form field gaps |
| lg | 24px | Card-to-card grid gap, section padding within page |
| xl | 32px | Page horizontal padding (desktop), major content block separation |
| 2xl | 48px | Login page vertical centering margin, empty state vertical padding |
| 3xl | 64px | Page-level top/bottom breathing room |

**Exceptions:**
- Tier badge: `px-4 py-2` (16px / 8px) — matched to hero element prominence; fits the 4px grid
- Touch targets (quick-add buttons in empty state): minimum 44px height — WCAG 2.5.5 requirement

## Typography

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` | Company name on card, last report date, form labels |
| Label | 12px | 400 (regular) | 1.4 | `text-xs` | "Last report:" prefix text, meta labels |
| Heading | 20px | 700 (bold) | 1.2 | `text-xl font-bold` | Ticker symbol on card (e.g., "VNM"), page headings ("Your Watchlist") |
| Display | 28px | 700 (bold) | 1.1 | `text-2xl font-bold` | Tier badge text inside the hero badge element |

**Font stack:** Inter, system-ui, -apple-system, sans-serif (shadcn/ui default; no Google Fonts import to keep CSP clean and reduce TTFB).

**Weight constraint:** Exactly 2 weights in use — 400 (body/label) and 700 (headings/badge). Both Heading and Display use 700 for strong visual hierarchy.

## Color

### Semantic Roles

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` — `--background` | Page background, card interiors |
| Secondary (30%) | `hsl(210 40% 96%)` — `--muted` | Card borders, skeleton shimmer base, input backgrounds |
| Accent (10%) | `hsl(222.2 47.4% 11.2%)` — foreground-strong | Primary CTA button ("Sign in"), quick-add buttons, link color |
| Destructive | `hsl(0 72.2% 50.6%)` — `--destructive` | Not used in Phase 12 (no delete/destructive actions in this phase) |

**Accent reserved for:** Sign in button, quick-add ticker buttons in empty state, Retry button in error state. Never used for tier badges, sparklines, or general card elements.

### Tier Badge Colors (locked in CONTEXT.md)

| Tier | Background | Text | Border | Tailwind Classes |
|------|-----------|------|--------|-----------------|
| Favorable | `teal-100` | `teal-800` | `teal-200` | `bg-teal-100 text-teal-800 border-teal-200` |
| Neutral | `slate-100` | `slate-700` | `slate-200` | `bg-slate-100 text-slate-700 border-slate-200` |
| Cautious | `amber-100` | `amber-800` | `amber-200` | `bg-amber-100 text-amber-800 border-amber-200` |
| Avoid | `rose-100` | `rose-800` | `rose-200` | `bg-rose-100 text-rose-800 border-rose-200` |

Dark mode variants appended: `dark:bg-teal-900/30 dark:text-teal-300` etc. per RESEARCH.md Pattern 5.

### Sparkline Colors (locked in CONTEXT.md)

| State | Hex | Tailwind Equivalent | Condition |
|-------|-----|---------------------|-----------|
| Price up year-over-year | `#16a34a` | `green-600` | `close[last] >= close[first]` |
| Price down year-over-year | `#dc2626` | `red-600` | `close[last] < close[first]` |

**Rationale for green/red sparkline:** The sparkline is a directional indicator, not a tier signal. Traffic-light green/red is appropriate here because it encodes a simple price direction fact, not an investment quality assessment. The tier badge (teal/slate/amber/rose) is the quality signal.

## Layout Contract

### Login Page (`/login`)
- Centered card layout: single column, max-width 400px, vertically centered on viewport
- Card contains: app name/logo (text only — "Stratum"), email input, password input, Sign in button
- No sign-up link (public signup is disabled — CONTEXT.md AUTH decision)
- No "forgot password" link in Phase 12 (invite-only; admin resets via Supabase dashboard)
- Error display: inline text below the form (not toast) for auth errors ("Invalid email or password")

### Dashboard Page (`/`)
- Full-width layout, no sidebar
- Page heading: "Your Watchlist" — `text-xl font-bold`
- Card grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6`
- Card max-width: unconstrained (fills grid column)
- Card internal layout: `p-4 space-y-3` with items stacked top-to-bottom per locked visual order

### Ticker Card Internal Order
1. **Symbol + company name** — `text-xl font-bold` for symbol, `text-sm text-muted-foreground` for name
2. **Tier badge** — centered horizontally, `text-2xl font-bold px-4 py-2`, full badge height
3. **Sparkline** — full card width, height 40px, SVG polyline
4. **Last report date** — `text-xs text-muted-foreground`, e.g. "Last report: 15 Mar 2026"

### Report Placeholder Page (`/reports/[symbol]`)
- Minimal scaffold only: heading with symbol name, "Full report coming in Phase 14" notice
- Back button to dashboard

## Interaction Contract

### Card Click
- **Target:** Entire card surface is clickable (not just a button within the card)
- **Behavior:** Navigate to `/reports/{symbol}` via Next.js `<Link>` wrapper on card
- **Hover state:** `hover:shadow-md hover:border-foreground/20 transition-shadow` — subtle elevation, no color shift
- **Focus state:** `focus-visible:ring-2 focus-visible:ring-ring` — keyboard navigable

### Quick-Add Buttons (Empty State)
- **Tickers shown:** VNM, FPT, HPG, GLD, MWG (5 buttons, all displayed)
- **On click:** Calls PUT /watchlist with the single ticker appended to current list (optimistic update: button transitions to disabled/loading state immediately)
- **After success:** Dashboard re-fetches and renders the new card
- **On failure:** Toast "Failed to add {symbol}" — button re-enables

### Sign In Button
- **Default state:** `variant="default"` — full accent color
- **Loading state:** Button disabled + spinner icon (Lucide `Loader2` with `animate-spin`) + text "Signing in..."
- **Error state:** Button re-enables; inline error text appears below form

### Retry Button (Error State)
- **On click:** Clears error state, resets loading to true, re-triggers the fetch lifecycle
- **Position:** Centered below error message text

### Toast (Sonner)
- **Position:** Bottom-right (Sonner default)
- **Duration:** 4000ms
- **Types used in Phase 12:**
  - `toast.error("Failed to load dashboard")` — first load failure
  - `toast.error("Refresh failed — showing cached data")` — subsequent refresh failure
  - `toast.error("Failed to add {symbol}")` — quick-add failure

## States and Transitions

### Dashboard States

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | Initial mount, before fetch completes | `WatchlistGridSkeleton` — 6 shimmer cards in responsive grid |
| Empty | Watchlist returns `{ tickers: [] }` | `EmptyState` — heading + body + 5 quick-add buttons |
| Populated | Fetch succeeds with 1+ tickers | `WatchlistGrid` with one `TickerCard` per ticker |
| Error (first load) | Fetch throws, `tickers` is still `[]` | Centered `ErrorState` — message + Retry button; no cards |
| Error (stale) | Refresh fails after initial success | `WatchlistGrid` still visible + toast "Refresh failed" |

### Card States

| State | Visual |
|-------|--------|
| No report yet | Tier badge shows "—" in `slate` style; last report date shows "No reports yet" |
| OHLCV unavailable | Sparkline area is empty (no SVG rendered); no error shown on card |
| Normal | Full card with badge + sparkline + date |

| Element | Copy | Notes |
|---------|------|-------|
| Primary CTA — login | "Sign in" | Verb only — no noun needed; Supabase context is clear |
| Page title | "Your Watchlist" | Possessive, friendly |
| Empty state heading | "Your watchlist is empty" | Direct, factual |
| Empty state body | "Add tickers to start tracking entry quality." | Action-oriented; explains the value |
| Empty state prompt | "Start with a popular ticker:" | Introduces quick-add buttons |
| Loading state | (no text — skeleton only) | Skeleton conveys loading without text noise |
| Error state heading | "Couldn't load your watchlist" | Describes the problem without jargon |
| Error state body | "Check your connection and try again." | Actionable next step |
| Error state CTA | "Try again" | Verb phrase on Retry button |
| Toast — first load fail | "Failed to load dashboard" | Plain description |
| Toast — refresh fail | "Refresh failed — showing cached data" | Explains what the user sees + why |
| Toast — quick-add fail | "Couldn't add {SYMBOL}" | Interpolated symbol; brief |
| No report yet (card) | "No reports yet" | Neutral; not alarming |
| Last report date | "Last report: {date}" | "Last report" prefix always present |
| Auth error (inline) | "Invalid email or password" | Generic — never expose whether email exists |
| Report placeholder page | "Full report view coming soon." | Phase 14 scope note; not shown to users in production |

**Copywriting principles:**
- No marketing language — this is a research tool, not a consumer product
- English only in Phase 12 — bilingual toggle is Phase 14 (RVEW-03)
- All dates formatted with `Intl.DateTimeFormat('en-GB', { day: 'numeric', month: 'short', year: 'numeric' })` → "15 Mar 2026"

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Tier badge color is NOT the only indicator | Badge always includes text label ("Favorable", "Neutral", etc.) — color is supplemental |
| Sparkline is decorative | SVG has `aria-hidden="true"` — no alt text needed; it is not the primary signal |
| Interactive cards are keyboard navigable | Card wrapped in Next.js `<Link>` which renders `<a>` — native tab/enter behavior |
| Quick-add buttons have accessible labels | `aria-label="Add VNM to watchlist"` on each quick-add button |
| Loading state announced | `aria-live="polite"` region wraps dashboard content area; screen readers announced when content changes |
| Form inputs have labels | shadcn/ui `<Label>` with `htmlFor` matched to input `id` |
| Minimum touch target | Quick-add buttons: `min-h-[44px]` — WCAG 2.5.5 |

### Peer Dependency Warning
If `npm install` produces `ERESOLVE` errors with React 19 peer deps, use `--legacy-peer-deps`:
```bash
npm install --legacy-peer-deps
```

### CSS Variables (globals.css)
shadcn/ui canary init writes CSS variables automatically. Tier badge colors (teal/slate/amber/rose) are Tailwind semantic classes — no additional CSS variable declaration needed beyond what init provides.

### No Dark Mode Implementation in Phase 12
Dark mode variants are declared on tier badge classes (per RESEARCH.md) but no dark mode toggle UI is built in Phase 12. The variants are there for when the system follows OS preference via `prefers-color-scheme`.
