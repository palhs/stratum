**1. Create `db/migrations/V8__watchlist.sql`:**

```sql
-- V8__watchlist.sql
-- Per-user watchlist and admin-editable default seed list

CREATE TABLE user_watchlist (
    id          BIGSERIAL   PRIMARY KEY,
    user_id     UUID        NOT NULL,
    symbol      VARCHAR(20) NOT NULL,
    asset_type  VARCHAR(20) NOT NULL
                    CHECK (asset_type IN ('equity', 'gold_etf')),
    added_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, symbol)
);

CREATE INDEX idx_user_watchlist_user ON user_watchlist (user_id);

CREATE TABLE watchlist_defaults (
    symbol      VARCHAR(20) PRIMARY KEY,
    asset_type  VARCHAR(20) NOT NULL
                    CHECK (asset_type IN ('equity', 'gold_etf')),
    sort_order  INTEGER     NOT NULL DEFAULT 0
);

-- Seed defaults: GLD + 5 core VN30 tickers
INSERT INTO watchlist_defaults (symbol, asset_type, sort_order) VALUES
    ('GLD',  'gold_etf', 0),
    ('VNM',  'equity',   1),
    ('VHM',  'equity',   2),
    ('VCB',  'equity',   3),
    ('HPG',  'equity',   4),
    ('MSN',  'equity',   5);
```

UNIQUE(user_id, symbol) prevents seeding race condition (duplicate tickers). Index on user_id for fast per-user queries.

**2. Rewrite `reasoning/app/auth.py` for RS256/JWKS:**

Replace the entire HS256 implementation with RS256/JWKS while keeping the same public interface (`require_auth` dependency returning decoded payload dict).

Key changes:
- Remove `import os` usage for SUPABASE_JWT_SECRET
- Add `from jwt import PyJWKClient` import
- Create module-level `_jwks_client` singleton: `PyJWKClient(os.environ["SUPABASE_JWKS_URL"], cache_jwk_set=True, lifespan=300)`
- In `require_auth`, replace `jwt.decode(..., secret, algorithms=["HS256"], ...)` with:
  1. `signing_key = _jwks_client.get_signing_key_from_jwt(cred.credentials)`
  2. `jwt.decode(cred.credentials, signing_key, algorithms=["RS256"], audience="authenticated")`
- Keep the same HTTPBearer(auto_error=False) pattern
- Keep the same exception hierarchy: ExpiredSignatureError→403, InvalidAudienceError→403, PyJWTError→401
- Keep the same docstring contract (401 missing/malformed, 403 expired/wrong-audience)

**CRITICAL:** The `_jwks_client` must be initialized at module level (NOT inside the function body) for caching to work. The 5-minute lifespan parameter means JWKS keys are fetched at most once per 5 minutes.
