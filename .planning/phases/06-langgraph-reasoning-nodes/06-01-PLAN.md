**Create reasoning/app/nodes/structure.py:**

The structure node is unique: it can use either a deterministic rules-based approach OR Gemini for narrative generation, but it MUST NOT recompute any values. It reads StructureMarkerRow from state and produces StructureOutput.

Implementation approach (Claude's discretion on tier assignment method):
1. Read `state["structure_marker_rows"]` — take the first row (or handle empty with NoDataError-like warning)
2. **Determine structure_label** using deterministic rules:
   - Compare close vs MA values (close > all 3 MAs → Constructive, close < all 3 MAs → Deteriorating, mixed → Neutral)
   - Factor in drawdown severity (drawdown_from_ath > 20% pushes toward Deteriorating)
   - Factor in percentile rank (close_pct_rank < 0.2 → Deteriorating bias)
3. **Generate narrative** using Gemini `with_structured_output(StructureOutput)`:
   - System prompt: "You are a price structure analyst. Interpret pre-computed technical markers into a narrative assessment. Do NOT compute or recalculate any values — use only the provided numbers."
   - Human message: formatted structure context from prompts.py
   - Pass the pre-determined structure_label as guidance in the prompt so Gemini's narrative is consistent with the label
4. Populate `sources` dict mapping each numeric field to source ID format: `"structure_markers:{symbol}:{data_as_of}"`
5. Propagate warnings from input StructureMarkerRow

**CRITICAL constraints per REAS-03 and SC #3:**
- NO import of retrieval functions (get_structure_markers, etc.) — only type imports
- NO recomputation of MAs, drawdowns, or percentiles
- Read ONLY from state["structure_marker_rows"]
