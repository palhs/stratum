**Create reasoning/app/nodes/entry_quality.py:**

1. Define structure veto logic:
```python
STRUCTURE_VETO_MAP = {
    "Deteriorating": "Cautious",  # Caps tier at Cautious
}
TIER_ORDER = ["Favorable", "Neutral", "Cautious", "Avoid"]
```

2. Define `_compute_base_tier(macro_label, valuation_label, structure_label) -> str`:
   - Use deterministic rules (Claude's discretion on method):
   - Map each sub-assessment to a score: Supportive/Attractive/Constructive=2, Mixed/Fair/Neutral=1, Headwind/Stretched/Deteriorating=0
   - Sum scores: 5-6=Favorable, 3-4=Neutral, 1-2=Cautious, 0=Avoid
   - This is a starting point; veto logic modifies it

3. Define `_apply_structure_veto(tier, structure_label) -> tuple[str, bool]`:
   - If structure_label in STRUCTURE_VETO_MAP: cap tier at the veto value
   - Return (final_tier, veto_was_applied)

4. Define `_apply_conflict_impact(tier, conflict_output) -> str`:
   - If conflict_output is None: no change
   - If conflict_output.severity == "major": downgrade tier by at least 1 level (Favorable→Neutral, etc.)
   - If conflict_output.severity == "minor": no automatic downgrade (Favorable can remain)

5. Define `entry_quality_node(state: ReportState) -> dict`:
   - Read all three node outputs from state: macro_regime_output, valuation_output, structure_output
   - Read conflict_output from state (may be None)
   - Extract labels: macro_label, valuation_label, structure_label
   - Compute base tier → apply structure veto → apply conflict impact
   - Check for stale data warnings across all inputs; if any exist, build stale_data_caveat string
   - Call Gemini `with_structured_output(EntryQualityOutput)` for narrative synthesis
   - Override LLM's composite_tier with the deterministically computed tier (LLM generates narrative only; tier is rule-based)
   - Return `{"entry_quality_output": result}`

**CRITICAL constraints per CONTEXT.md locked decisions:**
- Structure has VETO power — weak structure caps tier regardless of other signals
- Sub-assessments use domain-specific labels (not uniform 4-tier)
- No numeric score (anti-feature per out-of-scope)
- Stale data does NOT force Avoid — produce tier with caveat
- Conflict severity determines impact: minor allows Favorable, major forces downgrade
