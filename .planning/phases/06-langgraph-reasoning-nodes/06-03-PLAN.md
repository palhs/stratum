**Create reasoning/app/nodes/macro_regime.py:**

1. Define `macro_regime_node(state: ReportState) -> dict`
2. Read `state["fred_rows"]`, `state["regime_analogues"]`, `state["macro_docs"]`
3. Build prompt:
   - System prompt: "You are a macro regime analyst. Classify the current macroeconomic environment based on FRED indicator data and historical regime analogues. Output a probability distribution over regime types. Each regime probability must reference the specific analogue_id that supports it."
   - Human message: formatted FRED context + analogue context using prompts.py helpers
   - Include macro_docs content if available (FOMC/SBV document chunks for additional context)
4. Call Gemini `with_structured_output(MacroRegimeOutput)` — temperature=0.1 for consistency
5. Post-process the LLM response:
   - Sort regime_probabilities by confidence descending
   - Set top_regime_id and top_confidence from highest probability
   - Apply mixed-signal logic: `is_mixed_signal = (top_confidence < MIXED_SIGNAL_THRESHOLD)`
   - If is_mixed_signal: set mixed_signal_label = "Mixed Signal Environment", populate top_two_analogues with top 2 analogue IDs
   - Assign macro_label based on regime type: map regime classification to Supportive/Mixed/Headwind
6. Populate sources dict: "top_confidence" -> FRED source IDs, each RegimeProbability.source_analogue_id already serves as citation
7. Propagate warnings from input retrieval types
8. Return `{"macro_regime_output": result}`

**Key implementation detail:** The Gemini call should produce the regime probabilities and narrative, but the is_mixed_signal flag and threshold logic MUST be deterministic (post-LLM). Do not trust the LLM to correctly evaluate the < 0.70 threshold — compute it in Python after receiving the LLM's probability distribution.
