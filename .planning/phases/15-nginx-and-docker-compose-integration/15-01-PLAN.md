Create `nginx/` directory. Create two config files:

**nginx/local.conf** — HTTP only, for local development:
```nginx
upstream frontend_upstream {
    server frontend:3000;
    keepalive 16;
}

upstream reasoning_upstream {
    server reasoning-engine:8000;
    keepalive 16;
}

server {
    listen 80;
    server_name _;

    # SSE stream route — buffering disabled
    location ~ ^/api/reports/stream/ {
        proxy_pass http://reasoning_upstream;
        proxy_http_version 1.1;
        proxy_set_header Connection '';
        proxy_set_header Host $host;
        proxy_buffering off;
        proxy_cache off;
        gzip off;
        add_header X-Accel-Buffering no;
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;
    }

    # All other /api/* routes — normal buffered proxy
    location /api/ {
        proxy_pass http://reasoning_upstream;
        proxy_http_version 1.1;
        proxy_set_header Connection '';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Frontend — all other routes
    location / {
        proxy_pass http://frontend_upstream;
        proxy_http_version 1.1;
        proxy_set_header Connection '';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

**nginx/production.conf** — HTTPS with certbot ACME challenge and HTTP-to-HTTPS redirect. Use `DOMAIN_PLACEHOLDER` as the literal domain string (deployer replaces with their actual domain, or uses envsubst):
```nginx
upstream frontend_upstream {
    server frontend:3000;
    keepalive 16;
}

upstream reasoning_upstream {
    server reasoning-engine:8000;
    keepalive 16;
}

# HTTP: ACME challenge + redirect to HTTPS
server {
    listen 80;
    server_name DOMAIN_PLACEHOLDER;

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

# HTTPS
server {
    listen 443 ssl;
    server_name DOMAIN_PLACEHOLDER;

    ssl_certificate /etc/letsencrypt/live/DOMAIN_PLACEHOLDER/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/DOMAIN_PLACEHOLDER/privkey.pem;

    # SSE stream route — buffering disabled
    location ~ ^/api/reports/stream/ {
        proxy_pass http://reasoning_upstream;
        proxy_http_version 1.1;
        proxy_set_header Connection '';
        proxy_set_header Host $host;
        proxy_buffering off;
        proxy_cache off;
        gzip off;
        add_header X-Accel-Buffering no;
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;
    }

    # All other /api/* routes
    location /api/ {
        proxy_pass http://reasoning_upstream;
        proxy_http_version 1.1;
        proxy_set_header Connection '';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Frontend
    location / {
        proxy_pass http://frontend_upstream;
        proxy_http_version 1.1;
        proxy_set_header Connection '';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

Key details:
- SSE location block uses `~ ^/api/reports/stream/` regex to match the stream path specifically
- `proxy_buffering off` + `gzip off` + `proxy_cache off` ensures SSE events are not batched (per CONTEXT.md locked decision)
- `proxy_read_timeout 600s` on stream routes (10 min, per locked decision)
- `proxy_http_version 1.1` + `Connection ''` enables keepalive to upstream
- Normal `/api/` routes use default buffered proxy (per locked decision: SSE directives only on stream routes)
- `X-Forwarded-Proto $scheme` on API routes enables FastAPI to detect HTTPS
- Production conf has `DOMAIN_PLACEHOLDER` which the deployer replaces with their domain

**docker-compose.override.yml** — new file for local development (auto-merged by `docker compose up`):
```yaml
# Local development override — auto-merged by docker compose up
# Production: docker compose -f docker-compose.yml up -d (excludes this file)
services:
  nginx:
    volumes:
      - ./nginx/local.conf:/etc/nginx/conf.d/default.conf:ro
    ports:
      - "80:80"
    # Override: removes letsencrypt volumes, no port 443

  certbot:
    entrypoint: ["echo", "certbot disabled in local dev"]
    restart: "no"
```

Note on override behavior: the `volumes` key in the override REPLACES the base `volumes` list for nginx (this is Docker Compose merge behavior for sequences). The `ports` override also replaces base ports, giving us only port 80 locally (no 443).
