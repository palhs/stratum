    **Step 2: Create term_dict.py**
    - `load_term_dict() -> dict`: Load JSON from same directory using `pathlib.Path(__file__).parent / "term_dict_vi.json"`. Cache with module-level `_TERM_DICT` variable (loaded once on first call).
    - `apply_terms(report_json: dict) -> dict`: Create a deep copy of the report_json dict, then:
      - Replace `report_json["entry_quality"]["tier"]` using `tiers` lookup
      - Replace `report_json["macro_regime"]["label"]` using `macro_labels` lookup
      - Replace `report_json["valuation"]["label"]` using `valuation_labels` lookup
      - Replace `report_json["structure"]["label"]` using `structure_labels` lookup
      - Replace `report_json["conflict"]["pattern_name"]` using `conflict_patterns` lookup (if "conflict" key present and not None)
      - For each label replacement: use `dict.get(english_value, english_value)` — unknown labels pass through unchanged
      - Return the modified dict (new instance via copy.deepcopy, do not mutate input)
    - Note: apply_terms accepts a plain dict (the serialized report JSON), NOT a Pydantic model. This avoids a dependency on report_schema.py (Plan 02) and keeps Plan 03 parallel-safe in Wave 2.
    - Note: Narrative text is NOT replaced by apply_terms — narrative re-generation in Vietnamese is handled by compose_report_node (Plan 04) via a Gemini call. apply_terms only handles structured label fields.
