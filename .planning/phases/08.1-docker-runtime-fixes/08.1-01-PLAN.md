**Problem:** The Dockerfile build context is `./reasoning`. Currently `COPY app/ ./app/` places source at `/app/app/`. But Python imports use `from reasoning.app.*` which requires a `reasoning` package on sys.path. No such package exists in the container.

**Fix 1 — Restructure Dockerfile COPY to match import namespace:**

Replace the current Dockerfile with:

```dockerfile
FROM python:3.12-slim
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends curl \
    && rm -rf /var/lib/apt/lists/*
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
RUN mkdir -p reasoning && touch reasoning/__init__.py
COPY app/ ./reasoning/app/
COPY tests/ ./reasoning/tests/
COPY pytest.ini ./reasoning/
HEALTHCHECK --interval=15s --timeout=5s --retries=3 --start-period=60s \
    CMD curl -sf http://localhost:8000/health || exit 1
CMD ["uvicorn", "reasoning.app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

Key changes from current Dockerfile:
- `RUN mkdir -p reasoning && touch reasoning/__init__.py` — creates the `reasoning` Python package namespace
- `COPY app/ ./reasoning/app/` — places code at `/app/reasoning/app/` (was `/app/app/`)
- `COPY tests/ ./reasoning/tests/` — places tests at `/app/reasoning/tests/` (was `/app/tests/`)
- `COPY pytest.ini ./reasoning/` — moves pytest.ini into reasoning dir
- `CMD` uses `reasoning.app.main:app` instead of `app.main:app` — matches the absolute import namespace

Why this works: Python's cwd (`/app`) is on sys.path by default. `import reasoning` finds `/app/reasoning/` (a directory with `__init__.py`). `from reasoning.app.pipeline import generate_report` resolves to `/app/reasoning/app/pipeline/__init__.py`.

**Fix 2 — Add langgraph-init to reasoning-engine depends_on in docker-compose.yml:**

In the `reasoning-engine` service `depends_on` block (currently has postgres, neo4j, qdrant), add:

```yaml
      langgraph-init:
        condition: service_completed_successfully
```

Place it after the existing `qdrant` entry. This ensures checkpoint schema tables exist before reasoning-engine starts accepting requests.
