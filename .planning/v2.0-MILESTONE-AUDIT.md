## Cross-Phase Integration Issues

### BLOCKING: Import Path Mismatch in Docker Container

**Path:** `reasoning/Dockerfile` → `reasoning/app/pipeline/__init__.py` → `from reasoning.app.*` imports
**Affected requirements:** SRVC-01, REPT-01 through REPT-05, REAS-06

All pipeline and node modules use `from reasoning.app.*` absolute imports. The Dockerfile sets `WORKDIR /app` and copies `app/` into `/app/` — there is no `reasoning` package on the container's Python path. The lazy import in `_get_generate_report()` defers the failure from module load time, but `ModuleNotFoundError: No module named 'reasoning'` fires the first time a report job runs.

**Fix options:**
1. Add `ENV PYTHONPATH=/` to Dockerfile (simplest)
2. Restructure imports to `app.*` throughout
3. Copy `reasoning/` directory as-is to preserve package structure

### OPERATIONAL: Missing depends_on langgraph-init

**Path:** `docker-compose.yml reasoning-engine` → `docker-compose.yml langgraph-init`
**Affected requirements:** REAS-06, INFRA-06

`reasoning-engine` does not declare `depends_on: langgraph-init`. Both start under `profiles: ["reasoning"]`, but there's no ordering guarantee. First `run_graph()` call will fail if `langgraph-init` hasn't created the `langgraph` schema tables yet.

**Fix:** Add `langgraph-init: condition: service_completed_successfully` to `reasoning-engine.depends_on`.
