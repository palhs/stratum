## Cross-Phase Integration Issues

### Issue 1 (High): Earnings Freshness Silently Skipped

**Path:** `seed-qdrant-earnings-docs.py` → `qdrant_retriever.py search_earnings_docs()`
**Requirements affected:** RETR-04, DATA-04

The earnings seed script omits `document_date` from point payloads. The retriever's freshness guard reads `metadata.get("document_date")` → gets `None` → `if document_date_str:` skips → no warning ever emitted. Macro docs path works correctly (has `document_date` in payload).

**Fix:** Add `"document_date"` to earnings worker payload in `scripts/seed-qdrant-earnings-docs.py`.

### Issue 2 (Medium): Qdrant Host Default Mismatch

**Path:** `qdrant_retriever.py` → Docker reasoning network
**Requirements affected:** RETR-02

`qdrant_retriever.py` defaults `QDRANT_HOST` to `"localhost"` while all other components (seed scripts, conftest, docker-compose) default to `"qdrant"`. Will cause connection failure when Phase 6 reasoning-engine runs inside Docker.

**Fix:** Change default from `"localhost"` to `"qdrant"` in `qdrant_retriever.py` line 47.

### Issue 3 (Low): Source Validation Case Mismatch

**Path:** `seed-qdrant-macro-docs.py` validate function
**Requirements affected:** DATA-03

Payload stores `"fomc"`/`"sbv"` (lowercase) but validation queries for `"FOMC"`/`"SBV"` (uppercase). Validation always reports 0 points per source. Retrieval unaffected.

**Fix:** Change validation loop to lowercase in `seed-qdrant-macro-docs.py` line 510.
