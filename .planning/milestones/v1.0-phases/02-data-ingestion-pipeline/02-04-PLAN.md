**sidecar/app/services/pipeline_log_service.py:**

`log_pipeline_run(db_session, pipeline_name, status, rows_ingested, data_as_of, error_message=None, duration_ms=None)`:
- INSERT INTO pipeline_run_log with all fields
- This is a simple INSERT (not upsert) — every run creates a new log record
- data_as_of = the date of the data being ingested (pass-through from the endpoint)
- ingested_at = NOW() (database default)
- Return the created log record id

**sidecar/app/services/anomaly_service.py:**

`check_row_count_anomaly(db_session, pipeline_name, new_row_count)` → bool:
- Query last 4 successful runs for this pipeline from pipeline_run_log
- Compute average rows_ingested from those 4 runs
- If average is None or 0 (insufficient history), return False — skip check
- If abs(new_row_count - average) / average > 0.50, return True (anomaly detected)
- CRITICAL: This must NEVER raise an exception or block ingestion. Wrap in try/except, return False on any error.
- Anomaly detection is alert-only, NOT blocking (locked decision)

**Integration into ALL router endpoints:**

For EVERY endpoint (vnstock/ohlcv, vnstock/fundamentals, gold/fred-price, gold/gld-etf, gold/wgc-flows, fred/indicators, compute/structure-markers):

1. Record start time before calling service
2. Call the service function (existing code)
3. Record end time, compute duration_ms
4. Call log_pipeline_run() with appropriate pipeline_name (e.g., "vnstock_ohlcv", "gold_fred_price", "fred_indicators", "structure_markers")
5. For vnstock endpoints ONLY: call check_row_count_anomaly() and set anomaly_detected in the response
6. If service throws exception: log_pipeline_run(status="failure", error_message=str(e)), then re-raise

Use a decorator or context manager pattern to minimize boilerplate across endpoints. Example:

```python
import time
from contextlib import contextmanager

@contextmanager
def pipeline_run_context(db_session, pipeline_name, data_as_of):
    start = time.monotonic()
    try:
        yield
        # caller sets rows_ingested after yield
    except Exception as e:
        duration = int((time.monotonic() - start) * 1000)
        log_pipeline_run(db_session, pipeline_name, "failure", 0, data_as_of, str(e), duration)
        raise
```

Or alternatively, handle logging directly in each endpoint if a decorator adds too much complexity. The key requirement: EVERY endpoint writes to pipeline_run_log on EVERY run (success or failure).

**n8n/workflows/weekly-ingestion.json** — Weekly Data Pipeline:

Workflow structure:
```
Schedule Trigger (Sunday 2:00 AM Asia/Ho_Chi_Minh)
  → Set (compute date range: end_date=today, start_date=last week for incremental / 5 years ago for backfill)
  → HTTP Request: POST data-sidecar:8000/ingest/vnstock/ohlcv
  → [On Error → Retry Loop (3 attempts: 1min, 5min, 15min wait)]
  → HTTP Request: POST data-sidecar:8000/ingest/vnstock/fundamentals
  → [On Error → Retry Loop]
  → HTTP Request: POST data-sidecar:8000/ingest/gold/fred-price
  → [On Error → Retry Loop]
  → HTTP Request: POST data-sidecar:8000/ingest/gold/gld-etf
  → [On Error → Retry Loop]
  → HTTP Request: POST data-sidecar:8000/ingest/fred/indicators
  → [On Error → Retry Loop]
  → HTTP Request: POST data-sidecar:8000/compute/structure-markers
  → [On Error → Retry Loop]
  → IF (any anomaly_detected == true) → Telegram Alert (anomaly warning)
```

Key settings:
- Schedule Trigger: cron `0 2 * * 0` (Sunday 2:00 AM)
- Workflow Settings → Timezone: Asia/Ho_Chi_Minh (locked decision)
- Error Workflow: set to "Error Handler" workflow (error-handler.json)

**Custom retry loop pattern** (CRITICAL: n8n built-in retry caps at 5s delay — must use custom loop):
```
HTTP Request → [On Error] → Code (set attempt counter, delay) → Wait (delay seconds) → [Loop back to HTTP Request]
→ [After 3 failures] → Continue to next step (log failure, don't abort entire pipeline)
```

The Code node for retry delay:
```javascript
const attempt = ($input.first().json.attempt || 0) + 1;
const delays = [60, 300, 900]; // 1min, 5min, 15min
if (attempt > 3) {
  // Give up after 3 retries — move to error branch
  return [{ json: { giveUp: true, attempt } }];
}
return [{ json: { attempt, delay_seconds: delays[attempt - 1] } }];
```

Date computation in Set node:
```javascript
// For incremental weekly run:
const end_date = $now.toISODate(); // today
const start_date = $now.minus({ weeks: 2 }).toISODate(); // 2 weeks back (overlap for safety)

// For initial backfill (check if first run):
// If pipeline_run_log has no success records for this pipeline, use 5-year/10-year start dates
```

HTTP Request node settings (for each sidecar call):
- Method: POST
- URL: http://data-sidecar:8000/ingest/vnstock/ohlcv (etc.)
- Body content type: JSON
- Body: `{ "start_date": "{{ $json.start_date }}", "end_date": "{{ $json.end_date }}" }`
- Authentication: None (internal network)
- Timeout: 300000 (5 minutes — vnstock can be slow for backfill)
