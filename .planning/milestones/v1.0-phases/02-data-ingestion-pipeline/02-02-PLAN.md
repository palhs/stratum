**sidecar/app/services/gold_service.py:**

1. `fetch_and_upsert_gold_fred_price(start_date, end_date, db_session)`:
   - Use fredapi: `Fred(api_key=os.environ["FRED_API_KEY"]).get_series("GOLDAMGBD228NLBM", observation_start=start, observation_end=end)`
   - CRITICAL data_as_of semantics: The FRED `date` field IS the observation date (when gold price was fixed). Store this as `data_as_of`. Store NOW() as `ingested_at`. These are ALWAYS different.
   - Drop NaN values (FRED returns NaN for non-trading days)
   - Upsert to gold_price table with ON CONFLICT (source, data_as_of) DO UPDATE
   - Historical backfill: 10 years (locked decision) — caller passes start_date 10 years ago on first run
   - Return row count

2. `fetch_and_upsert_gld_etf(start_date, end_date, resolution, db_session)`:
   - Use yfinance: `yf.Ticker("GLD").history(start=start, end=end, interval="1wk", auto_adjust=True)`
   - Map columns: lowercase, date→data_as_of (normalize to midnight), add ingested_at=NOW()
   - Upsert to gold_etf_ohlcv with ON CONFLICT (ticker, resolution, data_as_of) DO UPDATE
   - Historical backfill: 10 years
   - Return row count

3. `fetch_and_upsert_wgc_flows(db_session)`:
   - WGC Goldhub is JS-rendered. Use one of these approaches (in order of preference):
     a. Check if WGC provides a direct Excel download URL for ETF flows (https://www.gold.org/goldhub/data/gold-etf-holdings-and-flows) — if the URL is static/predictable, download with httpx and parse with pandas
     b. If no direct download, use Playwright (must be added to requirements.txt and Dockerfile in this task if used) to open Goldhub, intercept network requests for the underlying JSON API, and capture the data
     c. Fallback: If WGC scraping proves unreliable during implementation, create a stub endpoint that returns 501 Not Implemented with a note that WGC data can be manually imported via CSV upload. Log this as a known limitation.
   - For central bank buying data: scrape from https://www.gold.org/goldhub/data/gold-demand-by-country (same approach as ETF flows)
   - Set data_as_of = period_end date (the reporting period end, NOT the scraping date)
   - Populate source_lag_note with "~45 day publication lag" for all WGC records
   - Upsert to gold_wgc_flows
   - WGC scraping runs monthly (locked decision) — not every weekly run

**sidecar/app/routers/gold.py:**

- POST /ingest/gold/fred-price — accepts IngestRequest(start_date, end_date), calls gold_service.fetch_and_upsert_gold_fred_price(), returns IngestResponse
- POST /ingest/gold/gld-etf — accepts IngestRequest(start_date, end_date, resolution default "weekly"), calls gold_service.fetch_and_upsert_gld_etf(), returns IngestResponse
- POST /ingest/gold/wgc-flows — no date params needed (always fetches latest available), calls gold_service.fetch_and_upsert_wgc_flows(), returns IngestResponse

**Register in main.py:** Add gold router with prefix="/ingest/gold", tags=["gold"]

If Playwright is used for WGC:
- Add `playwright>=1.47.0` to requirements.txt
- Add to Dockerfile: `RUN playwright install --with-deps chromium` after pip install
- Use async Playwright with network request interception

CRITICAL anti-patterns to avoid:
- Do NOT store the ingestion timestamp as data_as_of for FRED gold price — FRED date = observation date
- Do NOT use requests for WGC Goldhub — it's JS-rendered, returns empty HTML
- Do NOT scrape WGC weekly — monthly cadence only (locked decision, matches quarterly publication)

**sidecar/app/services/fred_service.py:**

1. Define FRED_SERIES constant:
```python
FRED_SERIES = {
    "GDP": {"frequency": "quarterly", "description": "US GDP"},
    "CPIAUCSL": {"frequency": "monthly", "description": "US CPI (Consumer Price Index)"},
    "UNRATE": {"frequency": "monthly", "description": "US Unemployment Rate"},
    "FEDFUNDS": {"frequency": "monthly", "description": "Federal Funds Effective Rate"},
}
```

2. `fetch_and_upsert_fred_indicators(start_date, end_date, series_ids, db_session)`:
   - If series_ids is None, fetch all FRED_SERIES keys
   - For each series: `Fred(api_key=os.environ["FRED_API_KEY"]).get_series(series_id, observation_start=start, observation_end=end)`
   - CRITICAL: The FRED `date` field on each observation = the **period the data covers**. For GDP date="2024-10-01" = Q4 2024. For UNRATE date="2024-12-01" = December 2024. This is `data_as_of`, NOT the API call date.
   - Build DataFrame with columns: series_id, value, frequency (from FRED_SERIES dict), data_as_of (FRED date), ingested_at (NOW())
   - Drop NaN values (FRED returns "." for missing observations)
   - Upsert to fred_indicators with ON CONFLICT (series_id, data_as_of) DO UPDATE
   - Return total row count across all series

**sidecar/app/routers/fred.py:**

- Pydantic model: FredIngestRequest(start_date, end_date, series_ids optional list[str])
- POST /ingest/fred/indicators — accepts FredIngestRequest, calls fred_service.fetch_and_upsert_fred_indicators(), returns IngestResponse
- Error handling: if FRED_API_KEY not set, return 503 with clear message

**Register in main.py:** Add fred router with prefix="/ingest/fred", tags=["fred"]
