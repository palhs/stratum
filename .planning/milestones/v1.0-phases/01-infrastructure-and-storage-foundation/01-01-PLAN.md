**Services — Storage tier (both networks, profiles: ["storage", "ingestion", "reasoning"]):**

1. `postgres` — `postgres:16-alpine`
   - restart: unless-stopped
   - Environment: POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB from .env
   - Volume: postgres_data:/var/lib/postgresql/data
   - Health check: `pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}` (interval: 10s, timeout: 5s, retries: 5, start_period: 30s)
   - NO port mapping (internal only — locked decision)
   - Networks: ingestion, reasoning

2. `neo4j` — `neo4j:5.26.21` (pinned — from RESEARCH.md)
   - restart: unless-stopped
   - Environment:
     - NEO4J_AUTH: neo4j/${NEO4J_PASSWORD}
     - NEO4J_PLUGINS: '["apoc"]'
     - NEO4J_server_memory_heap_initial__size: "512m"
     - NEO4J_server_memory_heap_max__size: "1G"
     - NEO4J_server_memory_pagecache_size: "512m" (critical for 8GB VPS — Pitfall 6)
     - NEO4J_apoc_trigger_enabled: "true"
   - Volumes: neo4j_data:/data, neo4j_logs:/logs
   - Ports: "7474:7474" (Browser), "7687:7687" (Bolt) — admin UI, exposed on host per locked decision
   - Health check: `wget --no-verbose --tries=1 --spider localhost:7474 || exit 1` (interval: 30s, timeout: 10s, retries: 5, start_period: 60s)
   - Networks: ingestion, reasoning

3. `qdrant` — `qdrant/qdrant:v1.15.3` (pinned — from RESEARCH.md)
   - restart: unless-stopped
   - Environment: QDRANT__SERVICE__API_KEY from .env
   - Volume: qdrant_storage:/qdrant/storage (MUST be named volume, not bind mount — Pitfall 7)
   - Health check: `curl -f http://localhost:6333/healthz` (interval: 10s, timeout: 5s, retries: 5)
   - NO port mapping (internal only — locked decision)
   - Networks: ingestion, reasoning

**Services — Migration/Init tier (one-shot, profiles: ["storage", "ingestion", "reasoning"]):**

4. `flyway` — `flyway/flyway:10` (pin to `10` series)
   - depends_on: postgres (condition: service_healthy)
   - Volume: ./db/migrations:/flyway/sql
   - Environment: FLYWAY_URL=jdbc:postgresql://postgres:5432/${POSTGRES_DB}, FLYWAY_USER, FLYWAY_PASSWORD
   - entrypoint: ["flyway", "migrate"]
   - Network: ingestion (only needs postgres access)
   - No restart policy (one-shot)

5. `neo4j-init` — `neo4j:5.26.21` (same image as neo4j for cypher-shell)
   - depends_on: neo4j (condition: service_healthy)
   - Volume: ./neo4j/init:/init-scripts
   - entrypoint: >
       bash -c "
         for f in /init-scripts/*.cypher; do
           echo \"Running $$f...\";
           cypher-shell -a bolt://neo4j:7687 -u neo4j -p $$NEO4J_PASSWORD -d system -f $$f;
         done;
         echo 'Neo4j init complete.'
       "
   - Environment: NEO4J_PASSWORD from .env
   - Network: ingestion (only needs neo4j access)
   - No restart policy (one-shot)
   - Note: Runs against system database (-d system) because APOC trigger.install() requires it in Neo4j 5.x

6. `qdrant-init` — `curlimages/curl:latest` (lightweight, only needs curl)
   - depends_on: qdrant (condition: service_healthy)
   - Volume: ./scripts/init-qdrant.sh:/init-qdrant.sh:ro
   - entrypoint: ["sh", "/init-qdrant.sh"]
   - Environment: QDRANT_API_KEY, QDRANT_HOST=qdrant, QDRANT_PORT=6333 from .env
   - Network: ingestion (only needs qdrant access)
   - No restart policy (one-shot)

   - Environment:

     - DB_POSTGRESDB_HOST: postgres
     - DB_POSTGRESDB_PORT: 5432

   - Volume: n8n_data:/home/node/.n8n
   - Port: "5678:5678" (n8n UI — exposed on host per locked decision)
   - Network: ingestion ONLY (INFRA-02 enforcement)
